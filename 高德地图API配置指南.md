# 高德地图API配置指南

## 问题描述
当前项目使用的高德地图API密钥无效，导致地图功能无法正常工作。

## 解决方案

### 1. 获取高德地图API密钥

1. **访问高德开放平台**
   - 网址：https://lbs.amap.com/
   - 注册/登录账号

2. **创建应用**
   - 进入控制台 → 应用管理 → 我的应用
   - 点击"创建新应用"
   - 应用类型选择"Web服务"

3. **添加Key**
   - 在应用详情中点击"添加"按钮
   - 服务平台选择"Web服务"
   - 获取生成的API Key

### 2. 配置环境变量

在 `web-app/.env` 文件中，将以下配置替换为真实的API密钥：

```env
# 高德地图API配置
VITE_AMAP_API_KEY=你的真实API密钥
VITE_AMAP_API_URL=https://restapi.amap.com
```

### 3. 当前临时解决方案

如果暂时无法获取真实API密钥，项目会进入"测试模式"：
- 地图显示为模拟界面
- 基本UI功能正常
- 实际地图功能受限

### 4. 功能对比

| 功能 | 测试模式 | 真实API模式 |
|------|---------|-------------|
| 地图显示 | 模拟界面 | 真实地图 |
| 路线规划 | 不可用 | 可用 |
| 地点搜索 | 不可用 | 可用 |
| 地理编码 | 不可用 | 可用 |
| 定位功能 | 不可用 | 可用 |

### 5. 验证配置

配置完成后，重启开发服务器：
```bash
cd web-app
npm run dev
```

访问应用，检查控制台是否还有API密钥错误。

## 注意事项

1. **安全性**：不要将API密钥提交到公开仓库
2. **配额限制**：高德地图API有每日调用限额
3. **域名绑定**：生产环境需要绑定域名
4. **服务开通**：确保相关服务（如地理编码、路径规划）已开通

## 技术支持

如果遇到配置问题，可以参考：
- 高德开放平台文档：https://lbs.amap.com/api/javascript-api/summary
- 项目文档中的API集成说明
- 联系技术支持获取帮助
